1. Explain in a few words your design decisions

I split the application in to different folders named after the page or section e.g. Home or Details. 

Each folder has an index.js which is the root component, as well as the necessary reducers, actions, sagas, tests etc. 

2. Which architectural patterns did you use in the past for web development if any?

Revealing module pattern, Singleton, MVC, Front controller, OOP
